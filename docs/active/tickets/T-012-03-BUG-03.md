## Summary

Chaos archetype (seed=42) clips into solid tile (6,31) at x=100, y=500-512 in Pipeworks
between frames 369-374, generating 8 `inside_solid_tile` invariant errors. The clipping is
brief (6 frames) but indicates a collision resolution failure near the stage start.

## Details

The Chaos archetype's random movement drives the player left from the start position (200,510)
to x=100 where it enters solid tile (6,31) at pixel coordinates (96,496)-(112,512).

Error frames:
```
f=369: center (100.0, 500.0) inside tile (6,31)
f=370: center (100.0, 506.2) inside tile (6,31)
f=371: center (100.0, 506.0) inside tile (6,31)
f=372: center (100.0, 512.5) inside tile (6,32)
f=373: center (100.0, 506.0) inside tile (6,31)
f=374: center (100.0, 512.8) inside tile (6,32)
```

Two additional clips occur later:
- f=1007: center (302.0, 424.0) inside tile (18,26)
- f=1747: center (297.9, 429.0) inside tile (18,26)

Total: 8 inside_solid_tile errors.

## Impact

Minor compared to T-012-03-BUG-01 and T-012-03-BUG-02. The primary Chaos failure is
BUG-01 (slope wall blocking progress to min_x=800). This clipping is a secondary issue
that adds invariant errors.

## Reproduction

```python
from speednik.qa import BehaviorExpectation, run_audit, make_chaos
exp = BehaviorExpectation(
    name="pipeworks_chaos", stage="pipeworks", archetype="chaos",
    min_x_progress=800, max_deaths=3, require_goal=False,
    max_frames=3600, invariant_errors_ok=0,
)
findings, result = run_audit("pipeworks", make_chaos(42), exp)
inv_errors = [v for v in result.violations if v.severity == "error"]
# len(inv_errors) == 8, all inside_solid_tile
```
