## Finding

Chaos archetype (seed=42) drifts left to x=-49488 on Hillside Rush. This triggers 10526
`position_x_negative` invariant errors.

## Evidence

- Chaos (seed=42) max_x=64.0, final_x=-49488.2 after 3600 frames
- 10526 invariant errors of type `position_x_negative`
- Player is never killed or stopped by the left boundary
- Random inputs generate net leftward drift, and nothing prevents escape

## Expected

Player should be clamped at x≥0 (or x≥some small margin). At minimum, physics should
prevent the player from moving into negative X territory.

## Reproduction

```python
from speednik.qa import run_audit, format_findings, BehaviorExpectation, make_chaos

exp = BehaviorExpectation(
    name="hillside_chaos", stage="hillside", archetype="chaos",
    min_x_progress=1200, max_deaths=2, require_goal=False,
    max_frames=3600, invariant_errors_ok=0,
)
findings, result = run_audit("hillside", make_chaos(42), exp)
print(format_findings(findings))
```

## Probable cause

No position clamping in `player_update` or `sim_step` for the left edge of the level.
The player freely moves into negative X coordinates without any collision or boundary
enforcement.
