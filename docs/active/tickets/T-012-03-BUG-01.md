## Summary

Angle=64 wall tiles at tile column 32 in Pipeworks form an impassable barrier atop a steep
slope, blocking all jumping/dashing archetypes before x=520. The slope approach tiles
(angle=32, type=3) reduce horizontal momentum, and the wall tiles at the top prevent further
progress.

## Affected Archetypes

| Archetype    | Expected min_x | Actual max_x | Blocked at        |
|--------------|---------------|-------------|-------------------|
| Jumper       | 5400          | 518.0       | tile (32, 11-12)  |
| Speed Demon  | 5400          | 449.0       | tile (~28, 17)    |
| Cautious     | 1500          | 447.2       | slope approach    |
| Chaos (s=42) | 800           | 428.9       | slope approach    |

## Root Cause

Tile column 32, rows 10-15 have angle=64 (wall quadrant) with solidity=2 (FULL):

```
tile(32,10): angle=64, solidity=2, type=1
tile(32,11): angle=64, solidity=2, type=1
tile(32,12): angle=64, solidity=2, type=1, heights[0:4]=[1,2,2,4]
tile(32,13): angle=64, solidity=2, type=1
tile(32,14): angle=64, solidity=2, type=1
tile(32,15): angle=64, solidity=2, type=1
```

The approach slope (columns 25-31) uses angle=32 tiles with type=3 (pipe terrain):

```
tile(25,19): angle=32, sol=2, type=3, heights=[0,2,2,4]
tile(26,18): angle=32, sol=2, type=3, heights=[1,1,3,3]
tile(27,17): angle=32, sol=2, type=3, heights=[1,2,2,4]
...up to tile(31,13): angle=32, sol=2, type=3
```

## Similarity to T-012-02-BUG-01

Hillside had a single angle=64 tile at (37,38) that blocked Walker/Cautious/WallHugger.
Pipeworks has an entire column of angle=64 tiles forming a taller wall. Same root cause
pattern â€” incorrect wall angle on terrain that should be traversable.
